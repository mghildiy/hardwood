# Contributing

Hardwood is an open-source project and contributions are welcome — from bug reports and documentation fixes to new features and performance improvements.

## Report a bug or request a feature

File an issue on the [GitHub issue tracker](https://github.com/hardwood-hq/hardwood/issues). Please check first whether a similar issue already exists.

## Contribute code or documentation

A good place to start is the curated list of entry points:

- [`good first issue`](https://github.com/hardwood-hq/hardwood/labels/good%20first%20issue) — small, self-contained tasks suitable for a first contribution.
- [`help wanted`](https://github.com/hardwood-hq/hardwood/labels/help%20wanted) — bounded work that welcomes pull requests.

The [CONTRIBUTING.md](https://github.com/hardwood-hq/hardwood/blob/main/CONTRIBUTING.md) guide covers the full workflow: issue-first process, build and test expectations, commit message format, and documentation requirements for user-facing changes.

## Ask a question or discuss an idea

For open-ended questions, design discussions, or show-and-tell, use [GitHub Discussions](https://github.com/hardwood-hq/hardwood/discussions). The issue tracker is reserved for concrete, actionable items.

## What's planned

The [roadmap](https://github.com/hardwood-hq/hardwood/blob/main/ROADMAP.md) lists the larger features in flight and on the horizon. If you're considering a sizable contribution, it's worth checking there — and opening an issue to align before starting — so your effort matches the project's direction.
